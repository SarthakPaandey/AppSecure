# CWE-798: Use of Hard-coded Credentials

**Official URL:** https://cwe.mitre.org/data/definitions/798.html

## Description

The software contains hard-coded credentials, such as a password or cryptographic key, which it uses for its own inbound authentication, outbound communication to external components, or encryption of internal data.

## Mitigations

- Never embed API keys or secrets in client-side JavaScript.
- Use server-side proxies for third-party APIs; store secrets in a secrets manager.
- Rotate any credentials that were ever exposed in bundles or repos.
- Scan CI artifacts for accidental secret inclusion.
