# A10:2021 – Server-Side Request Forgery (SSRF)

**Official URL:** https://owasp.org/Top10/A10_2021-Server-Side_Request_Forgery_%28SSRF%29/

SSRF flaws occur whenever a web application is fetching a remote resource without validating the user-supplied URL. It allows an attacker to coerce the application to send a crafted request to an unexpected destination, including internal services and cloud metadata endpoints (e.g., 169.254.169.254).

## Impact

- Scan/internal port scan from the application's perspective
- Access internal services not exposed to the internet
- Leak cloud credentials via metadata services
- Bypass firewalls and ACLs

## Related CWEs

CWE-918 (Server-Side Request Forgery).

## Prevention

- Prefer deny-by-default network policies for outbound application traffic
- Validate and sanitize all client-supplied URL data
- Allowlist DNS name / IP destination where possible
- Block requests to private/link-local IP ranges and metadata IPs
- Do not send raw responses to clients
- Disable HTTP redirections or validate redirect targets
