# A07:2021 – Identification and Authentication Failures

**Official URL:** https://owasp.org/Top10/A07_2021-Identification_and_Authentication_Failures/

Previously Broken Authentication. Confirmation of the user's identity, authentication, and session management is critical to protect against authentication-related attacks.

## Common issues

- Permits brute force or other automated attacks (missing rate limiting)
- Permits weak or well-known passwords; weak password policies
- Credential stuffing without detection
- Missing or ineffective multi-factor authentication
- Session identifiers exposed in URL; session fixation
- JWT "none" algorithm accepted; improper signature verification
- Secrets and API keys hardcoded in client-side code

## Related CWEs

CWE-287 (Improper Authentication), CWE-307 (Improper Restriction of Excessive Authentication Attempts), CWE-521 (Weak Password Requirements), CWE-798 (Use of Hard-coded Credentials).

## Prevention

- Implement MFA where possible
- Do not ship with default credentials
- Implement weak-password checks and breached password lists
- Align password length/complexity with modern guidance (prefer longer passphrases)
- Limit or increasingly delay failed login attempts; log and alert
- Use server-side secure session managers; regenerate session IDs on login
- Explicitly allowlist JWT algorithms; reject `alg=none`
- Keep secrets server-side only
