# A08:2021 – Software and Data Integrity Failures

**Official URL:** https://owasp.org/Top10/A08_2021-Software_and_Data_Integrity_Failures/

Related to code and infrastructure that does not protect against integrity violations. Includes insecure CI/CD, auto-update without integrity verification, and insecure deserialization.

## Common issues

- Mass assignment / auto-binding of attacker-controlled fields to internal objects (CWE-915)
- Insecure deserialization
- Untrusted plugins, modules, or CDN assets without integrity checks
- CI/CD pipeline without proper separation and signing

## Prevention

- Use digital signatures or similar to verify software/data integrity
- Ensure libraries and dependencies are from trusted repos
- Review code and configuration changes for supply-chain risks
- Do not auto-deserialize untrusted data
- Allowlist fields for mass assignment / use DTOs that exclude privileged attributes (e.g., role)
