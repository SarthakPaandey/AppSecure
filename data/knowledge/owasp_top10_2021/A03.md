# A03:2021 – Injection

**Official URL:** https://owasp.org/Top10/A03_2021-Injection/

Injection flaws (SQL, NoSQL, OS command, LDAP, XSS as a form of injection into interpreters) occur when untrusted data is sent to an interpreter as part of a command or query.

## Common issues

- User-supplied data is not validated, filtered, or sanitized
- Dynamic queries concatenated with user input without parameterization
- Hostile data used within ORM search parameters without safeguards
- Reflected or stored XSS when data is rendered without encoding

## Related CWEs

CWE-79 (XSS), CWE-89 (SQL Injection), CWE-77/78 (Command Injection), and others.

## Prevention

- Use safe APIs: parameterized queries, prepared statements, ORMs that escape by default
- Prefer positive server-side input validation (allowlists)
- Escape special characters for the destination interpreter
- For XSS: context-aware output encoding; Content-Security-Policy
- Use LIMIT and other SQL controls to prevent mass disclosure in case of SQLi
