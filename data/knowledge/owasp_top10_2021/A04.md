# A04:2021 – Insecure Design

**Official URL:** https://owasp.org/Top10/A04_2021-Insecure_Design/

A new category focusing on risks related to design and architectural flaws. Secure design is a culture and methodology that constantly evaluates threats and ensures code is robustly designed and tested.

## Common issues

- Missing or ineffective control design (e.g., no abuse-case modeling)
- Overly permissive workflows (e.g., unrestricted file upload without business justification controls)
- Trusting client-side enforcement for business rules
- Verbose error messages that leak internal design and secrets

## Prevention

- Establish secure development lifecycle with threat modeling
- Use ready-made secure design patterns and reference architectures
- Integrate security language and controls into user stories
- Limit resource consumption per user/service; fail securely
- Design for least privilege and fail-closed error handling
