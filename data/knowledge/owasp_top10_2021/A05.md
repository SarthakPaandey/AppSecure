# A05:2021 – Security Misconfiguration

**Official URL:** https://owasp.org/Top10/A05_2021-Security_Misconfiguration/

Security misconfiguration is commonly a result of insecure default configurations, incomplete setups, open cloud storage, misconfigured HTTP headers, and verbose error messages.

## Common issues

- Missing security headers (CSP, HSTS, X-Content-Type-Options, etc.)
- Unnecessary features enabled (directories, sample apps, GraphQL introspection in production)
- Default accounts and passwords unchanged
- Overly informative error handling revealing stack traces
- Out-of-date software and unpatched systems

## Prevention

- Repeatable hardening process; minimal platform without unnecessary features
- Automated verification of configuration in CI/CD
- Remove unused features and frameworks; don't install sample apps
- Send security directives (headers) to clients
- Disable GraphQL introspection and excessive debug modes in production
