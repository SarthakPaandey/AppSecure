# A02:2021 – Cryptographic Failures

**Official URL:** https://owasp.org/Top10/A02_2021-Cryptographic_Failures/

Previously known as Sensitive Data Exposure. Focuses on failures related to cryptography (or lack thereof) which often lead to exposure of sensitive data.

## Common issues

- Transmitting data in clear text (HTTP, SMTP, FTP)
- Weak or outdated cryptographic algorithms
- Default or weak crypto keys; keys committed to source control
- Missing encryption for sensitive data at rest
- Certificate validation disabled or incomplete (e.g., accepting self-signed certs without review)
- Use of deprecated hash functions for passwords

## Prevention

- Classify data and apply controls per sensitivity
- Encrypt data in transit with TLS; enforce HTTPS; enable HSTS
- Disable weak protocols (TLS 1.0/1.1, weak ciphers)
- Store passwords with strong adaptive salted hashing (Argon2, scrypt, bcrypt)
- Ensure proper certificate validation for outbound TLS (webhooks, APIs)
- Prefer authenticated encryption; manage keys securely
